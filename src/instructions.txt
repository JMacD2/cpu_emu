--- DATA ---
Enter in the form <ADDR>/<DATA>
THE ADDRESS SPACE IS 32-BIT - HEX ADDRESSES SHOULD BE 8 DIGITS LONG
THIS WILL BE LOADED FIRST
ADDRESSES ARE OF THE FORM 0x<xxxxxxxx> WHERE 'xxxxxxxx' IS SOME HEXEDECIMAL VALUE
NOTE -- INSTRUCTIONS WILL START FROM 0x00000000
NOTE -- ASCII TABLE IS STORED FROM 0xFFFFFF00 to 0xFFFFFF80
NOTE -- WRITE HEX DATA PREFACED BY 'Ox', DECIMAL PREFACED BY '0d', BINARY BY '0b'

--- INSTRUCTIONS ---
THIS WILL BE LOADED SECOND AND IN ORDER, STARTING FROM ADDR 0x00000000
THIS WILL THUS OVERWRITE DATA WHERE THERE IS A CONFLICT
THEY MUST BE WRITTEN IN A STANDARD FORM
FOR EXAMPLE:
ADD R1, R2, R3
SUB R1, R1, #5
B &00000001
LDR R1, &00000001
STR R1, &00000001
